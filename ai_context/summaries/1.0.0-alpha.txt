The 1.0.0-alpha release establishes an early foundation for managing AI context in Drupal. A single global context option simplifies demo and prototyping scenarios, while the Context Pools UI now links through to individual context details for easier review and editing. Admin navigation is grouped more logically, and the context listing shows human-readable descriptions instead of internal IDs.

Several reliability fixes improve how context is selected and delivered to agents, including correct handling of max token limits, required context tags, dependency injection for the function-call integration, and contexts configured via “always include.” Composer installation issues from incorrect dependency declarations are resolved. This alpha tagged the pre-rearchitecture codebase and sets the stage for the Context Control Center work that follows in beta1.
