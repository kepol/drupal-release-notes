Context Control Center (CCC) beta3 expands how context is authored, selected, and extended. New context source plugins let teams load context from PDFs, Markdown, plain text, and web pages, with a Document Loader submodule for richer document workflows. The context listing and item view pages are redesigned to show workflow state and align with updated UX patterns, while per-agent injection limits and selection modes give finer control over what each agent receives. A stable selection pipeline with extension events, optional feature suggestions in the UI, and test context items make CCC easier to adopt, customize, and validate.

This release continues RC1 preparation with stronger foundations and polish. Scheduler support moves toward progressive enhancement, bundleless context items simplify the content model, and AI Core 1.4 compatibility keeps CCC aligned with the broader AI ecosystem. Bug fixes improve scope form behavior, selection cacheability, admin UI alignment, and documentation accuracy. Security policy, GitLab templates, API documentation, and pre-release codebase review work further harden the module for production use.
