1.0.0-beta2

66 credited issues

Context Control Center (CCC) beta2 sharpens how context is scoped, delivered, and maintained as the module moves toward RC1. New entity type and bundle scope plugins, dynamically generated scope local tasks, and optional subcontext support give site builders finer control over which context applies where. Agent integrations are more reliable: full context is no longer re-injected on every agent loop iteration, programmatic context retrieval is easier for non-agent code, and admin links respect access checks. Progressive enhancement updates improve how target entities are handled in the UI. This release also focuses on stability and installability. Fixes address broken installs on existing configuration and recipe-based sites, fatal errors during uninstall, long context items that were skipped instead of truncated at token limits, and navigation and overview styling regressions. Under the hood, scope values are denormalized for better query performance, usage tracking is normalized and indexed, and global context item ordering is stabilized. Extensive UX review, Drupal CMS 2.1 compatibility testing, and codebase review work continue to harden CCC ahead of the first release candidate.
